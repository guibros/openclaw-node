# Arcane DevOps Checklist

## Before coding
- [ ] Read review/issue doc first.
- [ ] Confirm target repo path.
- [ ] Confirm branch strategy with user.

## Before commit
- [ ] Targeted tests pass.
- [ ] Full suite run and status recorded.
- [ ] Only intended files staged.
- [ ] Commit message reflects scope.

## Before push
- [ ] `git remote -v` confirms intended origin.
- [ ] Final suite green in destination repo.
- [ ] User instruction allows push target/branch.

## Final report
- [ ] What changed (files + intent)
- [ ] Test results (counts)
- [ ] Commit hash + push status
