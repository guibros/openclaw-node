# Desktop Notifications — ledgered, click-through, cross-platform

Every OpenClaw desktop popup is an event with a truth source and a way back to
its origin. The old path (fire-and-forget `terminal-notifier`, macOS-only, click
did nothing) is retired.

## The chain

```
emitter (workplan-viewer · plan ticks · node-watch · anything)
  │  bin/openclaw-notify.mjs  (or lib/notify.mjs directly)
  ▼
~/.openclaw/notifications/ledger.jsonl     ← append-only truth: EVERY event lands
  │                                          here, delivered or not (method
  │                                          recorded: terminal-notifier /
  │                                          osascript / notify-send / muted / none)
  ▼
OS popup, click-through to the origin URL
  macOS:  terminal-notifier -open <url> -contentImage <icon>
          (no terminal-notifier → osascript fallback: popup fires, honestly
           marked clickable:false in the ledger)
  Linux:  notify-send --icon <icon>; with libnotify ≥ 0.7.10 a detached waiter
          runs notify-send -A, and an activation → xdg-open <url> + a
          type:click line in the ledger
  ▼
origin: workplan-viewer (?plan=<id> deep link) · Mission Control /node-watch
browse history: Mission Control → Notifications (reads the ledger, shows
  kind/source/origin link/clicked state, filterable)
```

## Emitting

```sh
openclaw-notify --kind success --source my-daemon \
  --title "Thing happened" --message "details…" \
  --url "http://127.0.0.1:3000/node-watch"
openclaw-notify --list 20        # recent ledger entries
openclaw-notify --test           # one popup per kind
```

Kinds: `info` · `success` · `warn` · `error` · `block` (error/block are
`critical` urgency on Linux). Exit 0 = ledgered; `--strict` also requires the
popup to have been delivered. From Node, `import { notify } from 'lib/notify.mjs'`.

Wired emitters: workplan-viewer state transitions (step closed / plan BLOCKED,
deep-linking `?plan=<id>`), the tick-chain via the `memory-plan-notify.sh` shim,
and node-watch WORKING↔BROKEN transitions (linking to MC `/node-watch`).

## Ledger format

`~/.openclaw/notifications/ledger.jsonl` (override: `OPENCLAW_NOTIFY_LEDGER`):

```json
{"v":1,"type":"notification","id":"<uuid>","ts":"…","source":"workplan","kind":"success","title":"…","subtitle":"…","message":"…","url":"…","icon":"…","node":"<hostname>","delivery":{"method":"terminal-notifier","clickable":true,"ok":true}}
{"v":1,"type":"click","ref":"<uuid>","ts":"…"}
```

Click lines exist where clicks are observable (Linux action path). macOS
`-open` hands the URL straight to the OS — the click works but is not ledgered.

## Icons (the changeable terminal glyph)

The shipped set lives in `services/notify-icons/` (generated by
`generate-icons.mjs`, zero deps) and installs to `~/.openclaw/share/notify-icons/`.
Per-kind mapping in `~/.openclaw/config/notify.json`:

```json
{
  "enabled": true,
  "sources": { "node-watch": true },
  "icons": { "success": "party.png", "error": "/absolute/path/works/too.png" }
}
```

Drop any PNG in the icon dir and point a kind at it — next popup uses it. A
kind whose file is missing falls back to `default.png`. Per-event override:
`--icon`. Muting: `"enabled": false` (global) or `"sources": {"name": false}` —
muted events are still ledgered with `delivery.method: "muted"`.

## Fresh-node install

`install.sh` Step 16.5: copies icons, writes the default config, and checks the
platform dep — macOS `terminal-notifier` (brew-installed when brew is present),
Ubuntu `libnotify-bin` + `xdg-utils` (apt-installed). Degradation is honest and
recorded per event: no dependency → events still ledger, `delivery` says what
happened. No new daemon; the notifier is spawn-per-event.
