## 6. Revised decision tree

The HANDOFF's 8 decisions are still useful, but they need a Decision Zero in front:

### Decision 0 (new, must come first): Bridge the deploy gap

Before any other decision matters, you need a way to get the repo's code into the runtime tree without manual copy. Options:

- **0a.** Build the `bin/deploy-to-workspace.sh` from RESUME.md Block 8 Item F. Atomic diff/copy of `lib/*.mjs` + `bin/*.mjs` + `workspace-bin/*.mjs` from dev→workspace. Restart daemon. Verify healthy. ~half a day to write + test.

- **0b.** Change the launchd plist to point directly at `/Users/moltymac/openclaw-nodedev/workspace-bin/memory-daemon.mjs` and the daemon's imports at `/Users/moltymac/openclaw-nodedev/lib/*.mjs`. Eliminates the deploy step entirely. Simpler but means the dev repo IS the runtime — git status mid-edit could break the daemon.

- **0c.** Symlink `~/.openclaw/workspace/lib` → `/Users/moltymac/openclaw-nodedev/lib`. Same idea as 0b but via filesystem instead of plist edits.

**Recommendation: 0c** — fastest path to recovery. The plist already points at workspace/, so symlinking lib/ and bin/ (or just the daemon file) makes the repo the runtime. Then Decisions A-H from the HANDOFF become real.

Without Decision 0, every other "fix the daemon" decision in the HANDOFF is moot: the daemon is reading code from a different tree than the one you're editing.

## 7. Restart sequence (revised, smallest-possible)

If you pick Decision 0c + A1 + B1, this is the minimal viable restart:

1. **Verify NATS is running** — `ps aux | grep nats-server`. If not, the daemon's `mesh.memory.compaction_completed` subscription and the extract-trigger subscription are no-ops. Critical prerequisite.
2. **Stop the daemon** — `launchctl bootout gui/$(id -u)/ai.openclaw.memory-daemon`
3. **Replace deployed daemon + lib with repo symlinks (Decision 0c):**
   ```bash
   mv ~/.openclaw/workspace/bin/memory-daemon.mjs ~/.openclaw/workspace/bin/memory-daemon.mjs.may23
   mv ~/.openclaw/workspace/lib ~/.openclaw/workspace/lib.may23
   ln -s /Users/moltymac/openclaw-nodedev/workspace-bin/memory-daemon.mjs ~/.openclaw/workspace/bin/memory-daemon.mjs
   ln -s /Users/moltymac/openclaw-nodedev/lib ~/.openclaw/workspace/lib
   ```
4. **Restart daemon** — `launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/ai.openclaw.memory-daemon.plist`
5. **Tail `~/.openclaw/workspace/.tmp/memory-daemon.err`** — verify clean start with current code. If KeepAlive flaps, see error.
6. **Install consolidation-scheduler plist** — copy `services/launchd/ai.openclaw.consolidation-scheduler.plist` to `~/Library/LaunchAgents/`, run `launchctl bootstrap gui/$(id -u) ...`.
7. **Decision A1:** Wire federation factories into `workspace-bin/memory-daemon.mjs`. Port from `bin/openclaw-memory-daemon.mjs:38-240`. Single edit.
8. **Decision B1:** Wire `MemoryBudget.publishLocal` flow with a real local-event-log NATS stream verified to exist on disk.
9. **Fix `mentions.turn_index` properly** — change the extractor to emit per-turn citations (architectural change — flag as a Block 11+ step).
10. **Fix `skipIfExists` default** — set to `false`, or implement append-delta semantics.
11. **Fix `knowledge.db` auto-indexing** (Decision F1).
12. **Update `bin/openclaw-status.mjs`** — make it introspect the workspace daemon's actual wired components.

Steps 1-6 take an hour. Steps 7-12 are the actual feature work.
